literef_server module¶

This is the main module. It simply begins the =pyinotify= loop to listen to events from the drop folder. Namely, the server handles any file imported into LiteRef by dropping it into the drop folder and the request files created by the LiteRef command layer.

process_IN_CREATE(event)¶

This is a dispatcher. It passes the event to the proper handler. Right now this is very easy as we only need to handle appearance of a new file in the drop folder.

Parameters:event – is the event to be processed.
literef_server.init()¶

Creates the papers database, the survey and the drop folders unless they already exist. Also starts Tk.
