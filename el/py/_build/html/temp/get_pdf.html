get_pdf module¶

This module handles searching for PDF of a paper in the online sources given by the value of the variables config.PDF_AUTOMATED_SOURCES and config.BIB_AUTOMATED_SOURCES.

get_pdf.candidateBibFeedback(link, source)¶

Download the found candidate BibTeX entry and ask the user whether it is the one he desired.

Parameters:link – The direct link to the BibTeX entry.
Source:The online source.
Returns:The reply of the user (yes/no).
get_pdf.candidatePDFFeedback(link, paperDir)¶

Download the found candidate PDF and ask the user whether it is the one he desired.

Parameters:link – The direct link to the PDF.
PaperDir:The folder of the paper whose PDF is requested.
Returns:The reply of the user (yes/no).
get_pdf.confirmPDF(fileName)¶

Open the given PDF and ask the user whether is the one he wanted.

Parameters:fileName – The name of the PDF file.
Returns:The yes/no answer of the user.
get_pdf.entry2query(entry)¶

Compute a search query based on the given BibTeX entry or search string. If a BibTeX entry is supplied, then extract only the title. In both cases, replace spaces by pluses.

Parameters:entry – BibTeX entry or search string.
Returns:The computed search query.
get_pdf.firstLink(source, searchType, page)¶

Get the first link to the resource corresponding to the given search type (PDF of BibTeX) from the given HTML source page.

Parameters:
  • source – The online source.
  • searchType – The search type (“pdf” or “bib”).
Returns:

The first link to the resource being searched for. Returns an empty string if the afterLinkForPDF (or afterLinkForBib) method of the source (see the documentation of the online_sources module) raises an exception. Returns None If the link is not found on the page.
get_pdf.followRedirections(source, searchType, link)¶

Follow http re-directions beginning from the given link until a direct link to the searched for resource is obtained. Different sources and search types are handled appropriately.

Parameters:
  • source – The online source.
  • searchType – The search type (“pdf” or “bib”).
Returns:

The direct link to the searched for resource.
get_pdf.getResourceAutomated(entry, searchType, paperDir)¶

Search for the requested resource in the online sources given by the value of config.PDF_AUTOMATED_SOURCES or config.BIB_AUTOMATED_SOURCES depending on the search type.

Parameters:
  • entry – The paper’s BibTeX entry. This argument is provided for consistency of interface, but is not actually used.
  • searchType – The search type (“pdf” or “bib”).
  • paperDir – The folder containing the files associated with the paper.
Returns:

True if the user confirmed the found resource and False otherwise.
get_pdf.getResourceManual(entry, searchType)¶

Obtain the resource for the entry in a manual manner from the online source given by the value of config.PDF_MANUAL_SOURCE or config.BIB_MANUAL_SOURCE depending on the search type.

Parameters:
  • entry – The paper’s BibTeX entry. This argument is provided for consistency of interface, but is not actually used.
  • searchType – The search type (“pdf” or “bib”).
get_pdf.pdfExists(entry, paperDir)¶

Check whether the needed PDF file is among the PDF files in the drop folder.

Parameters:
  • entry – The paper’s BibTeX entry. This argument is provided for consistency of interface, but is not actually used.
  • paperDir – The folder containing the files associated with the paper.
Returns:

True if the paper PDF exists and False otherwise.
